## 4. Change Management

When a change to scope, schedule, or technical approach is needed:

1. **Identify** — Team member raises the change and its reason.
2. **Assess** — Evaluate impact on schedule, scope, and quality.
3. **Decide** — Team discusses and agrees to approve, defer, or reject.
4. **Implement** — If approved, update the plan and notify the team.

## 5. Corrective Actions

When progress deviates from the plan, the following corrective actions may be taken:

| Trigger | Corrective Action |
|---------|-------------------|
| Task overdue by > 2 days | Reassign resources or reduce task scope |
| Critical path task delayed | Fast-track by parallelizing dependent work |
| Test pass rate below 90% | Pause new development; focus on bug fixing |
| Team member blocked > 1 day | Escalate blocker in daily stand-up; reassign if unresolved |
| Scope creep identified | Defer non-essential additions to backlog |
